# 三套成功走路方案的可复现归档

本目录把学生机器人已经成功走起来的三种方法隔离保存。每套都包含训练代码快照、模型、机器人资产、Isaac/Isaac Gym 证据和 MuJoCo 闭环推理代码，避免以后只剩一个 `model_*.pt` 却无法还原当时的输入、动作和物理合同。

## 三套方法

| 目录 | 控制对象 | 方法 | 策略输入/输出 | 主要用途 |
|---|---|---|---|---|
| `01_xiaohai_r4_lowerbody12` | 学生 9.27 kg 原身，上半身固定，12 个腿关节 | 小海式周期步态奖励，不依赖外部参考动作 | 以归档代码实读为准 / 12 | 证明学生下半身在真实 25/10 Nm 分组下可以走 |
| `02_xingzhi_fullbody21` | 学生全身 21 关节 | 行止相位/速度命令强化学习，无外部动作文件 | 每帧 74 项 × 10 帧 = 740 / 21 | 已验证全身自由走路与冻结模型降力矩考试 |
| `03_reference_mimic_fullbody21` | 学生全身 21 关节 | 跟随由小海动作重定向得到的参考动作，同时学习保持平衡 | 每帧 75 项 × 10 帧 = 750 / 21 | 第三种独立成功路线；包含最终修正后的 MuJoCo 转换链 |

## 资产范围

- URDF：质量、惯量、关节轴、限位和网格引用。
- USD：Isaac/PhysX 导入后的资产层及配置层。
- MJCF/XML：MuJoCo 使用的模型；保留生成前和最终编译/评估版本。
- Mesh：URDF/USD/MJCF 引用的 STL 等几何文件。
- 模型：训练检查点及可用时的 TorchScript/ONNX 导出模型。
- 证据：Isaac 与 MuJoCo 的摘要、逐步数据和 H264/MP4 视频。

第一套 R4 当时由 Isaac Gym **直接导入 URDF**，并未以 USD 为训练输入。它目录中的 `fullbody_usd_reference_not_used_by_r4` 是同一学生全身资产的参考副本，不能误称为 R4 训练资产。

## 复现纪律

1. 先阅读对应子目录 README，确认关节顺序、观察量、动作映射、物理步长和策略周期。
2. 从归档资产运行回读门，不能只看 Python 配置。
3. Isaac 与 MuJoCo 必须使用同一关节顺序、输入历史、动作处理、PD、摩擦和策略周期。
4. 先加载随包模型复现保存的视频和摘要，再从零训练；不要同时修改奖励和物理合同。
5. `SHA256SUMS.json` 绑定归档内每个非缓存文件。复制或下载后执行：

   ```bash
   python tools/build_manifest.py --verify
   ```

详细的统一检查顺序见 [REPRODUCE.md](REPRODUCE.md)。这些方案是仿真可复现基线，不等于已经通过真机部署门。
