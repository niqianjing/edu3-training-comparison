# 统一复现流程

## 1. 固定环境

记录操作系统、Python、PyTorch、CUDA、Isaac/Isaac Lab/Isaac Gym、PhysX 和 MuJoCo 版本。原归档环境的主要版本为 Python 3.11.15、PyTorch 2.7.0+cu128、MuJoCo 3.3.3；R4 是较早的 Isaac Gym 工程，应优先使用其 `requirements.txt` 和独立环境，不能直接假设与后两套共用解释器。

## 2. 核对资产

逐项核对总质量、各刚体质量/质心/惯量、关节顺序、轴向、位置/速度/力矩限制、碰撞几何和材料。Isaac 导入后必须回读实际运行对象；旧 PhysX 无量纲 friction 字段应为零，实测 SI 摩擦由明确的执行器模型施加。

## 3. 先复现保存模型

先运行随包模型和评估器，确认能得到同类视频和摘要。若失败，先找合同差异，不要立即重训：

- 观察维数、排列和十帧历史是否一致；
- 历史保存的是原始动作还是实际执行动作；
- 关节排列和左右语义是否一致；
- 目标角是否经过相同的默认姿势、动作缩放和限位处理；
- 根部角速度是否取自相同坐标系；
- 参考动作是否按策略频率正确重采样；
- PD、摩擦、力矩限制、物理步长和策略周期是否一致。

## 4. 再从零训练

每次只改变随机种子或一个预先登记的变量。保留启动命令、配置快照、运行时回读、检查点和日志。两个种子不能被误称为两套不同方法，它们只是同一合同的重复实验。

## 5. 双引擎验收

固定前进命令和评估时长，关闭随机化、自动复位和额外辅助。至少报告是否摔倒、前进/侧向位移、转向、最低高度、最大倾角、目标越界、逐关节力矩 RMS/峰值/饱和率和连续额定比例。

## 6. MuJoCo 转换检查

MuJoCo 不是把策略文件“直接播放”就完成。必须逐项对齐：

1. 资产和关节顺序；
2. 观察量的内容、单位、坐标系和历史初始化；
3. 原始动作到目标角的映射；
4. 实际执行动作写回历史；
5. PD、显式摩擦、力矩限制和延迟；
6. 策略周期与物理子步；
7. 参考动作的帧率重采样。

第三套最终修复还包括：自由根角速度使用 MuJoCo `qvel[3:6]`，30 Hz、245 帧的参考动作重采样到策略 50 Hz、406 帧。任何一项没对齐，都可能出现 Isaac 能走而 MuJoCo 疯狂翻滚的假失败。
