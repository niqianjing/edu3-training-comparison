# 学生机器人两套训练代码对照（2026-07-21）

本目录用于让 Fable、Codex 与项目成员复核两条互不混用的训练路线。

## 目录

- `01_外部Edu3Flat原始包/`：用户提供的完整外部发布包，按原目录保留源码、缓存文件、URDF、MJCF、USD、网格、验证工具、校验清单与原说明，便于按原哈希复核。
- `02_Codex小海步态迁移/`：Codex 在墨造服务器上实际运行的“小海步态迁移到学生 9.27 kg 原身”代码快照，包括训练配置、任务注册、基础控制器、严格判卷、MuJoCo 实体网格复考脚本及两份实际 URDF。

## 两条路线不是同一合同

| 项目 | 外部 Edu3-Flat | Codex 小海迁移 R4 |
|---|---|---|
| 自由度 | 全身 21 关节 | 下肢 12 关节执行，上身固定参与质量与惯量 |
| 学习方式 | 相位纯强化学习，无参考步态 | 沿用小海相位步态任务与奖励，迁移到学生原身 |
| 策略周期 | 20 ms | 20 ms |
| 步态周期 | 完整周期 0.50 s | 半步 0.30 s，完整周期 0.60 s |
| 动作缩放 | 0.25 rad，动作裁剪 ±1 | 目标角由现有 HiController 合同生成 |
| 电机力矩 | 能力探针：髋前后/膝 50 Nm，髋侧摆/扭转/腰 20 Nm，踝两轴 40 Nm | 硬件候选：髋前后/侧摆/膝 25 Nm，其余下肢轴 10 Nm |
| 摩擦 | 显式 SI 摩擦，含 2–8 步随机延迟 | 显式 SI 摩擦，当前名义世界不做随机化 |
| 当前用途 | 验证外部方案是否能在其自身合同下学会走路 | 检查接近现有学生电机配置的可行走性 |

因此外部路线即使走得很好，也不能直接证明学生真机能承受 50/40 Nm 的训练动作；两者必须分别判卷，再分析差异来自算法还是电机合同。

## 已知 Codex R4 结果

- seed42 与 seed43 均通过 Isaac 和 MuJoCo 8 秒复考，没有摔倒或转圈。
- seed43 的 MuJoCo 结果：前进 1.112 m、侧移 0.103 m、转向 0.27°，髋侧摆持续负荷约额定的 72%/80%，无峰值饱和。
- 学生踝输出侧实测范围为前后摆约 `[-27°, +33°]`、左右摆约 `[-30°, +28°]`；训练 URDF 使用保守的 ±25°。

## 复核纪律

1. 不把两套任务的奖励、电机能力或步态周期混成一个结论。
2. 先做资产与运行时回读，再点火训练。
3. Isaac 与 MuJoCo 必须使用同一策略周期、目标角映射、电机 PD、摩擦和限位合同。
4. “仿真会走”与“具备真机条件”分开判定。
